# Декларация использования инструментов искусственного интеллекта

## Использованные инструменты

В ходе подготовки настоящей курсовой работы автор использовал следующие инструменты искусственного интеллекта на основе больших языковых моделей:

- **Claude** (компания Anthropic) — версии Sonnet 4.6, Opus 4.7 (с контекстным окном 1 миллион токенов), Haiku 4.5 — через интерфейсы Claude Code (приложение командной строки и расширение для редактора VS Code) и через веб-приложение Claude.ai.
- Других инструментов искусственного интеллекта (GPT, Gemini, Copilot и т. п.) в работе не использовалось.

## Задачи, в которых применялся искусственный интеллект

Инструмент применялся как ассистент в следующих категориях задач.

**(1) Программная реализация.** Инструмент использовался для написания скриптов на языке Python в следующих задачах:

- сбор данных с веб-сайтов Basketball-Reference и Hoopshype с обходом фильтрации Cloudflare;
- операции слияния между блоками данных с проверками целостности;
- реализация эконометрических спецификаций на основе пакетов `statsmodels` и `linearmodels`;
- построение декомпозиции $R^2$ методом значения Шепли с генерацией всех $2^9 = 512$ подмножеств блоков;
- кластерный и дикий бутстрап на кластерах, а также спецификация событийного исследования;
- генерация визуализаций средствами библиотек `matplotlib` и `seaborn`.

Весь сгенерированный код был запущен, а его выводы (коэффициенты, p-значения, $R^2$) сопоставлены с независимыми расчётами на языке R, выполненными на исходной структуре данных, и с теоретическими ожиданиями (аксиома эффективности для метода Шепли, тест воспроизводимости с хеш-снимком исходной структуры). Регрессионный тест воспроизводимости подтверждает идентичность численных результатов независимым расчётам с точностью до $5 \times 10^{-5}$.

**(2) Литературное оформление.** Подготовка черновиков аналитических заметок, разделов глав, аннотации и итогового заключения. Все черновики были вычитаны автором; стиль изложения, структура аргументов, выбор формулировок и финальная редактура остаются за автором.

**(3) Структурирование гипотез.** Синтез финального набора из десяти гипотез (H1–H10) из четырёх исходных гипотез предыдущей итерации проекта и четырёх новых гипотез настоящей итерации, с группировкой по пяти направлениям исследования. Инструмент помог в формальном изложении гипотез и в их сопоставлении с конкретными эконометрическими спецификациями; концептуальная связь гипотез с литературой и тематическим вопросом работы установлена автором.

**(4) Документирование.** Заполнение отчётных markdown-файлов, генерация таблиц в формате CSV, оформление подписей к рисункам.

## Задачи, в которых искусственный интеллект НЕ участвовал

- **Постановка исследовательского вопроса.** Тема и исследовательский вопрос работы сформулированы автором до начала использования инструментов искусственного интеллекта.
- **Выбор источников данных и временного периода.** Решения о работе с источниками Basketball-Reference и Hoopshype, о выборе сезонов с 2015/16 по 2023/24, о структуре семи новых блоков данных приняты автором на основе предварительного плана исследования.
- **Содержательная интерпретация результатов.** Все экономические интерпретации в Главе 5 (Обсуждение) и Главе 6 (Заключение) являются оригинальными авторскими выводами, основанными на знании литературы. Инструмент помогал в литературном оформлении изложения, но не в концептуальной интерпретации.
- **Методологические решения.** Выбор метода Шепли вместо последовательной декомпозиции, решение использовать дикий бутстрап на кластерах, выбор анализа Oster (2019) для проверки чувствительности к пропущенным переменным приняты автором на основе изучения соответствующей литературы.
- **Подбор и оценка цитируемых источников.** Список литературы и оценка релевантности каждого источника к конкретным гипотезам выполнены автором при сотрудничестве с коллегой (см. репозиторий, каталог `bibliography/`).

## Верификация результатов, сгенерированных с участием искусственного интеллекта

Применены следующие процедуры независимой проверки.

1. **Регрессионный тест воспроизводимости базовой структуры.** Запуск базовых спецификаций (модели M1a–M1d) на расширенном наборе данных с использованием только переменных исходной структуры показывает максимальное расхождение коэффициентов с прямыми оценками на исходной структуре не более $5 \times 10^{-5}$. Такой тест исключает возможность того, что инструмент мог гипотетически ввести артефакты в базовые соотношения.

2. **Аксиоматическая проверка декомпозиции Шепли.** Аксиома эффективности $\sum_i \phi_i = R^2_{\text{full}}$ верифицирована численно до машинной точности (расхождение $1{,}4 \times 10^{-16}$). Выполнение этого математического равенства исключает реализационно-зависимые ошибки.

3. **Хеш-снимок наборов данных.** Криптографический хеш SHA-256 исходной структуры панели зафиксирован в файле `analysis_v2/reports/v1_snapshot.sha256` и проверяется при каждой пересборке проекта.

4. **Размер выборки и степени свободы.** Для каждой регрессионной таблицы число наблюдений и регрессоров проверено и согласуется с теоретическим расчётом (см. раздел 4.2 Главы 4).

5. **Параллельный расчёт ключевых коэффициентов на языке R.** Спецификации M1a, M1c и M2c независимо запущены в среде R (директория `scripts/*.R`) и сопоставлены с реализациями на Python.

6. **Сквозная проверка библиографических ссылок.** Все ссылки в драфтах глав 1–6 сверены с базой `bibliography/references.bib` через автоматизированную проверку (`bibliography/bib_check_report.md`); по результатам финальной проверки выявлено ноль пропущенных записей и ноль неверифицированных цитирований.

## Асинхронное сотрудничество двух сессий искусственного интеллекта

Работа подготовлена через асинхронное сотрудничество двух независимых сессий инструмента Claude Code на двух физических компьютерах с координацией через систему контроля версий git.

- **Сессия Claude-K** на стороне автора работала над главами курсовой (`analysis_v2/reports/chapter_*.md`), методологическим приложением, аннотацией, декларацией использования искусственного интеллекта, сборкой манускрипта и конвертацией через Pandoc в формат .docx.
- **Сессия Claude-A** на стороне коллеги-помощника работала над библиографией (каталог `bibliography/sources/`, файлы `references.bib`, `MASTER_TABLE.md`), извлечением содержательных выводов из верифицированных PDF-документов и сквозной проверкой цитирований.

Координация осуществлялась через git-основанный протокол с явной дисциплиной разделения файлов между сессиями (без пересечений в правах редактирования), формальной очередью задач (файл `coordination/TASKS.md`) и каналами межсессионного обмена сообщениями (файлы `coordination/FOR_ARTEM.md` и `coordination/FOR_KAROLINA.md`). Все 50 с лишним задач распределены между двумя сессиями; изменения, влияющие на содержательную часть работы, документированы в файле `bibliography/findings_log.md` с указанием временной метки и обоснования.

Полный атомарный журнал всех коммитов обеих сессий содержится в файле `coordination/ai_disclosure_commits_log.md`. Для каждого коммита фиксируются: идентификатор сессии (Claude-K или Claude-A), временная метка, текст коммита и изменения через стандартную команду `git show`. Рецензенты могут независимо аудитировать вклад каждой сессии через стандартные команды git (см. файл `coordination/reproducibility_appendix.md`, раздел C.2).

## Открытый исходный код

В целях обеспечения воспроизводимости результатов и прозрачности использования инструментов искусственного интеллекта весь исходный код, сырые данные, промежуточные таблицы и черновики в формате markdown размещены в публичном репозитории на GitHub:

**https://github.com/outofcost/CourseProject**

Репозиторий включает: полный программный конвейер от сбора данных до генерации финальных рисунков (пакет `analysis_v2/`); регрессионный тест воспроизводимости (`analysis_v2/regress_test_v1.py`); хеш-снимки наборов данных; контекстный файл `CLAUDE.md` для инструментов искусственного интеллекта; черновики всех аналитических заметок в формате markdown.

Любой исследователь может воспроизвести все численные результаты работы примерно за три минуты при наличии прогретого кэша или примерно за два часа при «холодном» запуске со сбором данных с нуля.
